# Service Outage Response Guide

## Summary

When customers report a possible outage, support agents should avoid confirming a platform-wide incident until the internal status source or engineering team confirms it.

## Standard Handling

Agents should collect the affected product area, customer region, error messages, time of first impact, browser or device details, and whether the issue affects one user or multiple users.

If an incident is confirmed, agents may acknowledge the issue and provide the approved status update. If no incident is confirmed, agents should continue troubleshooting using normal technical support steps.

## Customer-Safe Wording

"Thank you for reporting this. We are checking the affected service area and will share confirmed updates as soon as they are available. In the meantime, please send any error messages, timestamps, and affected user details so we can investigate further."

## Escalation

Escalate immediately if multiple customers report the same issue, a customer reports data loss, or the issue affects payment, login, or core service availability.
